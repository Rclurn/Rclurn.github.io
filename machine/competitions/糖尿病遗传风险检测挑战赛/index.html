数据分析

读取数据并初探数据

利用pandas对比赛所给的数据进行读取，这里由于数据里面有大量中文字符，所以编码方式应该为“gbk”

import pandas as pd
train_dataset = pd.read_csv('./data/糖尿病遗传风险预测挑战赛公开数据/比赛训练集.csv', encoding='gbk')
test_dataset = pd.read_csv('./data/糖尿病遗传风险预测挑战赛公开数据/比赛测试集.csv', encoding='gbk')

利用info()函数初步探查数据的整体情况(各字段类型、数据量大小、是否含有空值等)

train_dataset.info(verbose=True)

可以看到，训练集总共有5070条数据，测试集总共有1000条数据，其中只有“舒张压”特征含有空值。

数据分析

train_dataset['舒张压'] = train_dataset['舒张压'].fillna(train_dataset['舒张压'].mean())
test_dataset['舒张压'] = test_dataset['舒张压'].fillna(test_dataset['舒张压'].mean())

3、分析字段类型

字段类型的判断主要还是通过人为根据其实际意义来做出判断

字段类型
类别类型 性别、糖尿病家族史
数值类型 出生年份、体重指数、舒张压、口服耐糖量测试、胰岛素释放实验、肱三头肌皮褶厚度

类别类型的数据因为没有大小关系，所以可以考虑将其转换为one-hot编码，这里训练集中的"糖尿病家族史"值为中文汉字，对于许多算法都不支持该类型的数据，所以需要将其转换为数字型数据，类别型特征没有大小关系，转换为序列数字可能存在一定的大小关系，所以最终将其转换为one-hot编码形式的数据

train_dataset['糖尿病家族史'].unique() # 查看有多少个不同的类别
temp_dict = {'无记录':0, '父母有一方患有糖尿病':1, '叔叔或者姑姑有一方患有糖尿病':2, '叔叔或姑姑有一方患有糖尿病':3} # 依据类别将中文转换为对应的数字
train_dataset['糖尿病家族史'] = train_dataset['糖尿病家族史'].apply(lambda x: temp_dict[x])
df1 = pd.get_dummies(train_dataset['糖尿病家族史'], prefix='糖尿病家族史') # 转换为对应的one-hot编码
train_dataset = pd.concat([train_dataset, df1], axis=1) # 拼接数据

4、检查数据集中是否存在不可靠数据

train_dataset.describe()
train_dataset[train_dataset['体重指数']==0]
test_dataset[test_dataset['体重指数']==0]

体重指数存在为0的值，根据常识该值不可能为0，进一步发现体重指数为0的数据中大部分“胰岛素释放实验”、“肱三头肌皮褶厚度”均为0，这是不符合常理的，而且这样的数据数量不算太多，对于训练集，考虑将这样的样本删掉，测试集这样的样本只有2条，就原封保留。
